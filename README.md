# backend_fastapi_ecom_web


backend/models где само сабой описываются сами модели. Например обект продукта ам может быь название продукта и тогдали, И после написание моделей мы делаем миграцию это в свою очред создает папку миграция с помощью каторого потом в базе данных создаются таблицы 


backend/repositories, зачем он нужен кратка говоря он проста берет вещи с базы и возвращяет их и он на прямую работает с базами данных


backend/routers, это некий маршрутизатор там есть пти каторые отвечают за сервис каторые книму подключены и он проста туда перенаправляет


backend/schemas, это почти аналог serialize в django, в этой папке описывается формат для api приводет к нормальному виду в json для frontend и еще можно сказать то что он валидирует данные 


backend/services,  пака содерижил функций классы, какторые выполняю оновную бизнес-логику приложения это проста оброботка данных
